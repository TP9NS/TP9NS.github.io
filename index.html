트러블 슈팅

문제 1

사용자의 답변(영상)이 여러 개의 파일로 전송되며 처리 지연 발생.

해결: 사용자에게 필요한 다음 질문만 동기 처리하고, 나머지 분석은 비동기로 처리하여 Redis 버퍼에 저장 후 결과 요청 시 전송하도록 구조 개선

문제 2

면접 종료 후 모든 분석이 끝나지 않았는데 사용자가 결과를 요청하는 비동기 결과 집계 동기화 문제 발생.

해결: 면접 종료 시 웬소켓을 통해 서버간 연결을 유지하고, 각 분석 서버에서 완료 시 Redis에 Done 플래그를 저장하도록 개선

문제 3

배포 과정에서 백엔드 서버가 수평 확장 되다보니 마지막 분석 결과를 받은 서버와 웹소켓이 연결되어 있는 서버가 달라 이벤트 유실 발생

해결: Redis Sub/Pub을 도입해 모든 분석 완료 시 모든 백엔드 서버에 이벤트를 발행하고 웹소켓이 연결되어 있는 서버가 수신하여 클라이언트로 전송하도록 처리

문제 4

모바일 환경에서 소셜 로그인 시 OAuth2 리다이렉션이 모바일 브라우저 정책에 따라 차단되어 로그인 실패 발생.

해결: 브릿지 컨트롤러를 통해 플랫폼 (모바일, 웹)을 구분해 세션에 저장하고, 모바일의 경우 인증 완료 시 OTT를 accessToken과 rtid 형태로 임시 저장한 뒤, 앱으로 딥링크를 전달하여 토큰 교환 후 삭제되도록 구현.

트러블 슈팅

문제 1

루틴 추천 노드에서 사용자의 컨디션(수면, 피로도 등)을 고려하지 않고 운동을 추천하는 문제가 발생

해결: 수면이 운동 수행에 미치는 영향과 관련된 논문을 기반으로 벡터 DB를 구축하고, RAG를 통해 추천 로직에 반영하여 루틴을 제공하도록 개선

문제 2

초기에는 JWT 토큰을 SessionStorage에 저장했으나, 브라우저 종료 시 세션이 끊켜 사용자가 매번 재로그인해야 하는 불편함 발생

해결: 토큰 저장 방식을 HttpOnly Cookie로 변경하여 보안성과 지속성을 확보하고, 로그인이 유지되도록 개선
